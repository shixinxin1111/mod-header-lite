# Chrome 请求头插件页面设计说明

## 1. 页面目标

弹窗页面需要在有限空间内同时承载配置选择、请求头编辑与 URL 规则编辑三类核心任务，因此界面以“桌面工具面板”作为视觉方向，强调信息密度、清晰分区和快速操作。

## 2. 布局结构

整体尺寸目标为：

- 宽度：`720px`
- 高度：`560px`

页面采用三段式结构：

1. 顶部工具栏
2. 左侧配置列表
3. 右侧双区编辑面板

```text
+--------------------------------------------------------------+
| 顶部工具栏：全局开关 / 当前配置 / 状态提示                    |
+----------------------+---------------------------------------+
| 左侧配置列表         | 请求头编辑区                          |
|                      +---------------------------------------+
|                      | URL 规则编辑区                        |
+----------------------+---------------------------------------+
```

## 3. 视觉风格

- 主基调：深色工作台
- 强调色：清晰但克制的蓝色
- 信息层次：通过边框、轻阴影和透明度区分层级
- 字体策略：使用系统字体，保证中文显示稳定与清晰
- 图标策略：优先图标化操作，减少按钮长文案

## 4. 顶部工具栏

### 4.1 包含元素

- 插件标题
- 当前配置名称
- 全局启用开关
- 当前状态提示文案

### 4.2 交互要求

- 开关视觉必须明显
- 当插件停用时，仍允许查看和编辑配置，但要明确提示“当前不生效”
- 当前配置名称要在顶部清晰展示，帮助用户建立上下文

## 5. 左侧配置列表

### 5.1 包含元素

- 配置列表项
- 新增配置按钮
- 删除配置按钮

### 5.2 交互要求

- 当前启用配置具有明确高亮态
- 列表项紧凑显示，避免冗长说明
- 删除配置前进行二次确认
- 新增配置后自动切换到该配置

## 6. 请求头编辑区

### 6.1 列结构

- 请求头名称
- 请求头值
- 启用开关
- 删除操作

### 6.2 交互要求

- 支持新增空白行
- 支持行内编辑
- 支持单行启停
- 支持单行删除
- 当无数据时展示空状态提示

## 7. URL 规则编辑区

### 7.1 列结构

- 正则表达式输入框
- 启用开关
- 删除操作

### 7.2 交互要求

- 支持新增空白规则
- 支持行内编辑
- 支持单行启停
- 支持单行删除
- 非法正则需立即提示，并说明该规则不会参与生效

## 8. 组件建议

建议拆分为以下视图组件：

- `ProfileSidebar`
- `ProfileItem`
- `HeaderEditor`
- `UrlFilterEditor`
- `SectionCard`
- `EmptyState`

## 9. 可用性要求

- 所有可点击区域具备明确 hover 状态
- 关键输入区域支持清晰的 focus 态
- 删除操作使用图标按钮，并附带可访问性文本
- 区域滚动时维持内容边界清晰，避免弹窗显得拥挤
